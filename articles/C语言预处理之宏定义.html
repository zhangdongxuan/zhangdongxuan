C语言预处理之宏定义

0  评论    1,730  浏览

    C语言的预处理我只是在看别人的源码的时候看见过，也听别人说过但是就是不知道这到底是怎么回事，这次终于有机会揭开它那神秘的面纱了。

     

    C语言中的预处理指令有三种：

    1.宏定义

    2.条件编译

    3.文件包含

    1、所有的预处理指令都是以#开头
    2、预处理指令在代码翻译成0和1之前执行。
    3、预处理的位置是随便写的，但是要注意作用域

     

    首先说明一下宏定义，这是我第一次接触宏定义，宏定义分为两种：无参宏定义和有参宏定义。

    #define COUNT  6                         这是无参宏定义，其中COUNT是宏名一般写成大写，6是宏的值

    #define sum(v1,v2) v1+v2            这是带参数的宏定义，其实这个比较像函数，但是又不是函数

    看下面两段代码：

    1、

    #define sum(v1,v2) v1+v2

int main(){
	int c = sum(1,3);
	printf("%d\n",c);

	return 1;
}

    2、

    #define sum(v1,v2) v1+v2

int main(){
	int c = sum(1,3)*sum(2,8);
	printf("%d\n",c);

	return 1;
}

    我们在代码段1中输出的结果是4，那么看见代码段2的时候可能第一反应是会输出结果40，但是实际结果却是15。

    这是因为预处理指令是在代码翻译成0和1之前执行的，所以main函数在执行之前，

    int c = sum(1,3)*sum(2,8);
    被翻译成

    int c = 1+3*2+8;

    所以输出的结果是15，可见有参宏定义和函数还是不一样的，所以要想让它的结果是40我们应该这样声明：

    #define sum(v1,v2) ((v1)+(v2))

    添加了一个括号就可以保证他的运算顺序是按照我们的预期的，不过一定要记住预处理指令是在代码翻译成0和1之前执行的。在声明宏定义的时候参数要加小括号，返回结果也要加小括号。

    使用宏定义的优势：带参数的宏定义的效率比函数要高。
